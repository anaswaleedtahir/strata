# Frontend Architecture

Strata uses a Django-owned frontend pipeline. Django templates remain the page and layout layer, reusable markup lives in Cotton components, Tailwind CSS is built by `django-tailwind-cli`, and browser libraries that do not need local customization are loaded from pinned CDN URLs with SRI.

## Stack

- Tailwind CSS 4 through `django-tailwind-cli`
- Project-owned, stock shadcn-style primitives scaffolded into Django Cotton
- Django Cotton for reusable template components and domain components
- HTMX and Alpine.js from pinned CDN URLs in `templates/_layouts/base.html`
- WhiteNoise for collected static files

There is no Node.js project dependency, no npm build pipeline, and no local JavaScript bundler.

## Source And Output

```
assets/
└── css/
    ├── input.css          # Tailwind 4 CSS-first source and shadcn tokens
    └── tw-animate.css     # Vendored animation utilities

static/
├── dist/
│   └── output.css         # Generated Tailwind output
├── images/                # Static images
└── js/
    └── chat-client.js     # Browser-served chat client

staticfiles/               # Generated by collectstatic
```

`assets/css/input.css` is the source file. It is not collected directly by Django. `django-tailwind-cli` writes the compiled stylesheet to `static/dist/output.css`, and `collectstatic` copies that output into `staticfiles/`.

## Tailwind Configuration

Tailwind is configured in `config/settings/tailwind_cli.py`:

```python
TAILWIND_CLI_SRC_CSS = "assets/css/input.css"
TAILWIND_CLI_DIST_CSS = "dist/output.css"
TAILWIND_CLI_USE_DAISY_UI = False
TAILWIND_CLI_VERSION = "4.2.2"
TAILWIND_CLI_AUTOMATIC_MINIFY = True
```

The stock shadcn light and dark tokens live in `assets/css/input.css` using Tailwind 4 CSS-first configuration. The application currently renders the light theme; dark tokens are retained for a later switcher. Add reusable CSS in `@layer` blocks only when utilities are not enough.

## Build Process

Development:

```bash
just runserver
```

`just runserver` runs uvicorn with `--reload` and `python manage.py tailwind watch` in parallel. Use `just uvicorn` or `just tailwind-watch` to run either process alone.

Production:

```bash
uv run python manage.py tailwind build --force
uv run python manage.py collectstatic --noinput
```

Docker runs the same Tailwind build before `collectstatic`; the runtime image contains the virtualenv, application code, and collected static files, but no Node.js toolchain.

## Template Components

Reusable components use Cotton:

```
templates/cotton/
├── forms/
├── navigation/
└── ui/

apps/properties/templates/cotton/properties/
```

Use shared components from `templates/cotton/` for cross-app UI. Use app-level Cotton directories for domain-specific components. Keep page templates and layout templates as normal Django templates.

`shadcn_django` is a development-time scaffolder, not an installed runtime library. Its output is reviewed and committed as project-owned Cotton source. To scaffold another primitive without adding Node.js:

```bash
uvx shadcn_django@0.24.1 add <component>
```

Generated primitives must declare forwarded props such as `class` with `<c-vars>` so Cotton does not emit duplicate HTML attributes. Preserve the stock neutral shadcn tokens unless a new architecture decision changes the visual language.

Example:

```django
<c-forms.field :field="form.email" label="Email" placeholder="Enter your email" required />
<c-properties.property-card :property="property" />
```

## JavaScript

HTMX and Alpine.js are loaded from pinned CDN URLs with SRI in the base layout. Project-owned JavaScript that does not need bundling lives directly in `static/js/`.

The chat client is served as:

```django
{% load static %}
<script src="{% static 'js/chat-client.js' %}"></script>
```

## Troubleshooting

If CSS does not update, run:

```bash
uv run python manage.py tailwind build --force --verbose
```

If static files are missing, run:

```bash
uv run python manage.py collectstatic --noinput
```

If the Tailwind CLI binary is missing or stale, remove `.django_tailwind_cli/` and rerun the build command.
